# 資產保管權變動與租借流程設計

> 目標：修正目前保管人可隨意修改、借出中裝置缺乏歸還強制的問題，  
> 使資產持有鏈符合 ISO 27001 A.8 資產管理要求。

---

## 一、現況問題分析

### 1.1 保管人可任意修改，無稽核軌跡

| 問題 | 說明 |
|------|------|
| 欄位直接改 | 資產表單上 custodian 是 dropdown，直接選就換人 |
| 舊值被覆蓋 | `UPDATE assets SET custodian_id=..., custodian_name=...` 無歷史紀錄 |
| 無審批流程 | 保管權移轉不需要任何人核准 |
| 違反 A.8.1.2 | ISO 27001 要求資產擁有權的變更需有正式紀錄 |

### 1.2 「借用日期」(borrow_date) 語意不明

- `assets.borrow_date`：目前作為「保管人開始持有的日期」
- `rentals.borrow_date`：租借系統的借出時間
- 兩個欄位名稱相同但意義不同，容易混淆
- 資產表單可手動填寫 borrow_date，不受租借流程控制

### 1.3 借出中裝置的保管權缺口

```
目前可能的違規路徑：

保管人 A 持有裝置
  ↓ 租借系統借給 B（status=active）
  ↓ 此時 asset.custodian 仍然是 A 或被改成 B
  ↓ 有人直接修改 asset.custodian 改成 C
  ↓ B 歸還時，保管人已經不是 A 了

結果：責任鏈斷裂，不知道誰該為裝置負責
```

### 1.4 未強制「先歸還再借出」

- 前端 `AssetForm` 有 `isRented` 檢查，但只擋 UI
- 後端 `PUT /api/assets/:id` 可直接修改 custodian，無 rental 狀態檢查
- 租借建立時有 `CheckDeviceAvailability`，但保管人變更沒有

## 二、核心概念重新定義

### 2.1 角色定義

| 角色 | 說明 | 誰設定 | 可變更嗎 |
|------|------|--------|----------|
| **管理人** (owner) | 資產的行政負責單位/人，通常是 IT 部門 | 建立資產時指定 | 可改，需紀錄 |
| **保管人** (custodian) | 實際持有並負責保管資產的人 | 透過「領用」或「保管權移轉」流程 | 需走正式流程 |
| **借用人** (borrower) | 透過租借系統暫時持有資產的人 | 租借系統自動帶入 | 租借歸還後自動清除 |

### 2.2 資產持有狀態

```
┌──────────────────────────────────────────────────┐
│                    資產狀態                        │
├──────────┬───────────┬───────────────────────────┤
│ 狀態     │ 保管人     │ 目前持有人                 │
├──────────┼───────────┼───────────────────────────┤
│ 閒置     │ 無        │ 無（在庫房）                │
│ 已領用   │ A         │ A（保管人 = 持有人）        │
│ 借出中   │ A         │ B（借用人，租借系統追蹤）    │
│ 歸還     │ A         │ A（回到保管人手上）          │
│ 報廢     │ A（凍結）  │ 無                         │
│ 移撥     │ —（已移出）│ 無                         │
└──────────┴───────────┴───────────────────────────┘
```

**核心規則**：  
- 借出中的資產，保管人**不可變更**（後端強制，非僅前端）
- 裝置必須**先歸還**才能再借給下一個人
- 保管人變更必須產生 `custody_transfer` 稽核紀錄

## 三、欄位變更設計

### 3.1 `borrow_date` → `assigned_date`（領用日期）

將原本語意不明的 `borrow_date` 改為 `assigned_date`，代表**保管人領用此資產的日期**。

| 項目 | 舊 | 新 |
|------|----|----|
| DB 欄位 | `borrow_date DATE` | `assigned_date DATE` |
| Go 欄位 | `BorrowDate *time.Time` | `AssignedDate *time.Time` |
| 前端 label | 借用日期 | 領用日期 |
| 設定方式 | 手動填寫 | 保管權移轉時自動帶入，也可手動設定 |
| 意義 | 不明確 | 保管人開始對此資產負責的日期 |

### 3.2 新增 `asset_custody_logs` 表

記錄所有保管權變動的歷史。

```sql
CREATE TABLE IF NOT EXISTS asset_custody_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    asset_id        UUID NOT NULL REFERENCES assets(id) ON DELETE CASCADE,
    action          TEXT NOT NULL,  -- 'assign', 'transfer', 'return', 'revoke'
    from_user_id    UUID REFERENCES users(id),
    from_user_name  TEXT NOT NULL DEFAULT '',
    to_user_id      UUID REFERENCES users(id),
    to_user_name    TEXT NOT NULL DEFAULT '',
    reason          TEXT NOT NULL DEFAULT '',
    operated_by     UUID NOT NULL REFERENCES users(id),  -- 執行此操作的人
    operator_name   TEXT NOT NULL DEFAULT '',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_custody_logs_asset ON asset_custody_logs(asset_id);
CREATE INDEX idx_custody_logs_time  ON asset_custody_logs(created_at DESC);
```

### 3.3 assets 表欄位調整

```sql
-- 1. borrow_date → assigned_date
ALTER TABLE assets RENAME COLUMN borrow_date TO assigned_date;

-- 2. 新增 current_holder_* 欄位（由租借系統自動維護）
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_id   UUID REFERENCES users(id);
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_name TEXT NOT NULL DEFAULT '';
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_since TIMESTAMPTZ;
```

欄位語意：

| 欄位 | 說明 | 誰寫入 |
|------|------|--------|
| `custodian_id` / `custodian_name` | 保管人（長期負責人） | 保管權移轉 API |
| `assigned_date` | 保管人領用日期 | 保管權移轉時自動帶入 |
| `current_holder_id` / `current_holder_name` | 目前實際持有人 | 租借系統自動帶入/清除 |
| `current_holder_since` | 持有開始時間 | 租借啟動時自動帶入 |

## 四、流程設計

### 4.1 保管權指派 / 移轉

```
POST /api/assets-custody
{
  "action": "assign" | "transfer",
  "asset_id": "uuid",
  "to_user_id": "uuid",
  "reason": "新進設備分配" | "部門調動"
}
```

**前置檢查：**
1. 資產狀態不可為 `retired` / `transferred`
2. 資產不可處於借出中（`current_holder_id IS NOT NULL`）
3. 操作者需有 `asset:manager` 權限

**執行動作：**
1. 記錄舊保管人資訊
2. 更新 `assets.custodian_id`, `custodian_name`, `assigned_date = today`
3. 寫入 `asset_custody_logs`（action, from, to, reason, operator）
4. 寫入 `audit_logs`（module=asset, action=custody_assign / custody_transfer）

### 4.2 保管權收回

```
POST /api/assets-custody
{
  "action": "revoke",
  "asset_id": "uuid",
  "reason": "離職繳回"
}
```

**執行動作：**
1. 清除 `custodian_id`, `custodian_name`, `assigned_date`
2. 寫入 custody log + audit log

### 4.3 租借流程串接（修正）

#### 借出時（rental activate）
```
現有邏輯：
  rental.status = 'active'
  asset.asset_status = 'rented'
  asset.custodian_id = borrower_id  ← 問題：覆蓋了保管人

修正為：
  rental.status = 'active'
  asset.asset_status = 'rented'
  asset.current_holder_id = borrower_id      ← 寫入持有人，不動保管人
  asset.current_holder_name = borrower_name
  asset.current_holder_since = now()
```

#### 歸還時（rental return）
```
現有邏輯：
  rental.status = 'returned'
  asset.asset_status = 'available'
  asset.custodian_id = original_custodian  ← 需要記住原保管人

修正為：
  rental.status = 'returned'
  asset.asset_status = 'available'
  asset.current_holder_id = NULL      ← 清除持有人
  asset.current_holder_name = ''
  asset.current_holder_since = NULL
  # custodian_id 始終不動，不需要「記住」
```

#### 借出前置檢查（後端強制）

```go
// 建立租借時
func (c *RentalController) handleCreate(...) {
    for _, udid := range body.DeviceUdids {
        // 現有檢查
        assetStatus, isRented, isLostMode, _ := c.rentalRepo.CheckDeviceAvailability(ctx, udid)

        // 新增：必須有保管人才能借出
        // （可選：視業務需求決定是否強制）
    }
}

// 保管人變更時
func (c *AssetController) handleCustodyTransfer(...) {
    // 檢查是否借出中
    if asset.CurrentHolderID != nil {
        return error("資產目前借出中，需先歸還才能變更保管人")
    }
}
```

### 4.4 保管人修改保護（後端）

在 `PUT /api/assets/:id` 中，**移除** `custodian_id` 和 `custodian_name` 的直接修改權限。

```go
// asset_repo.go Update()
allowed := []string{
    "device_udid", "asset_number", "name", "spec", "quantity", "unit",
    "acquired_date", "unit_price", "purpose",
    // "borrow_date",      ← 移除（改為 assigned_date，由 custody API 管理）
    // "custodian_id",     ← 移除（必須走 custody API）
    // "custodian_name",   ← 移除
    "location", "asset_category", "notes", "category_id", "asset_status",
    "dispose_reason", "transferred_to",
}
```

## 七、DB Migration (016)

```sql
-- 016_custody_redesign.up.sql

-- 1. Rename borrow_date → assigned_date
ALTER TABLE assets RENAME COLUMN borrow_date TO assigned_date;

-- 2. Add current holder fields
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_id    UUID REFERENCES users(id);
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_name  TEXT NOT NULL DEFAULT '';
ALTER TABLE assets ADD COLUMN IF NOT EXISTS current_holder_since TIMESTAMPTZ;

-- 3. Custody transfer log table
CREATE TABLE IF NOT EXISTS asset_custody_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    asset_id        UUID NOT NULL REFERENCES assets(id) ON DELETE CASCADE,
    action          TEXT NOT NULL,
    from_user_id    UUID REFERENCES users(id),
    from_user_name  TEXT NOT NULL DEFAULT '',
    to_user_id      UUID REFERENCES users(id),
    to_user_name    TEXT NOT NULL DEFAULT '',
    reason          TEXT NOT NULL DEFAULT '',
    operated_by     UUID NOT NULL REFERENCES users(id),
    operator_name   TEXT NOT NULL DEFAULT '',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX IF NOT EXISTS idx_custody_logs_asset ON asset_custody_logs(asset_id);
CREATE INDEX IF NOT EXISTS idx_custody_logs_time  ON asset_custody_logs(created_at DESC);

-- 4. Backfill: for currently rented assets, move custodian → current_holder
UPDATE assets a
SET current_holder_id = a.custodian_id,
    current_holder_name = a.custodian_name,
    current_holder_since = a.updated_at
WHERE a.asset_status = 'rented'
  AND a.custodian_id IS NOT NULL;
```

## 九、ISO 27001 對照

| 條款 | 要求 | 本設計對應 |
|------|------|-----------|
| A.8.1.1 資產清冊 | 資產應被識別並維護清冊 | assets 表 + AssetList 頁面 |
| A.8.1.2 資產擁有權 | 每項資產應有指定的擁有者 | custodian_id（保管人）不可任意修改 |
| A.8.1.3 資產可接受使用 | 應識別、記錄使用規則 | 租借流程 + 保管權移轉流程 |
| A.8.1.4 資產歸還 | 員工離職時歸還資產 | custody revoke + assigned_date 清除 |
| A.8.2.3 資產處理 | 資產移轉/報廢需有紀錄 | asset_custody_logs + audit_logs |
